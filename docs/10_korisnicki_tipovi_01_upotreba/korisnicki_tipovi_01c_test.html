Набројиви типови и структуре - квиз¶

    Q-127: Које су све последице употребе набројивих типова (означи све тачне реченице)?

  • Употреба програма је олакшана корисницима.
  • Значење појединих аргумената у методима постаје јасније.
  • Програми постају краћи.
  • Теже је правити семантичке грешке (багове) у коду.

    Q-128: Константе набројивог типа се интерно региструју као … ?

  • бројеви израчунати на основу имена ових константи
  • Не.
  • стрингови
  • Не.
  • специјални кодови, који се увoде за сваки набројиви тип посебно
  • Не.
  • целобројне вредности
  • Тачно!

    Q-129: Означи све тачне реченице о пољима структуре.

  • Поља структуре морају да буду међусобно истог типа.
  • Поља структуре морају да буду простог типа (string, int, double и сл.).
  • Свако поље у структури заузима простор у меморији.
  • У наредбама ван структуре могу да се користе само јавна (public) поља структуре.
  • Структуре су референцирани тип података.

    Q-130: Означи све тачне реченице о својствима структуре.

  • Својство може да буде доступно само за читање, само за писање, или за читање и писање.
  • На месту употребе, својство изгледа потпуно исто као поље.
  • Ако својство има оба приступника (set и get), оно је исто што и поље.
  • Свака структура мора да има бар једно својство.

Шта се исписује извршавањем овог кода?

using System;

class Program
{
    struct Tacka { public double x; public double y; }

    public static void Main()
    {
        Tacka A = new Tacka { x = 1, y = 2 };
        Tacka B = A;
        A.x += 3;
        Console.WriteLine(B.x);
    }
}

Шта се исписује извршавањем овог кода?

using System;

class Program
{
    struct Tacka { public double x; public double y; }

    static void Pomeri(Tacka t, double dx, double dy)
    {
        t.x += dx;
        t.y += dy;
    }
    public static void Main()
    {
        Tacka A = new Tacka { x = 1, y = 2 };
        Pomeri(A, 3, 3);
        Console.WriteLine(A.x);
    }
}
